# دليل الحماية الأوفلاين + تشغيل وتسليم التطبيق للعميل

## الهدف
هذا المستند يشرح طريقة عملية لحماية تطبيق DeltaErp في وضع الأوفلاين، مع خطوات تسليم وتشغيل واضحة للعميل.

## النطاق
- تفعيل أوفلاين فقط بدون اعتماد دائم على الإنترنت.
- ربط الترخيص بجهاز محدد.
- استخدام ترخيص موقّع رقميا.
- إدارة نقل الترخيص لجهاز جديد بشكل منضبط.
- وجود خطوات تشغيل ودعم واضحة بعد التسليم.

## ملاحظة مهمة عن الوضع الحالي بالمشروع
شاشة تسجيل الدخول الحالية تحتوي بيانات تجريبية ثابتة داخل الملف lib/features/auth/presentation/login_page.dart.
هذا مناسب للتجربة فقط، لكنه غير كاف للحماية التجارية.
لازم التحقق من الترخيص يتم قبل السماح بالدخول للتطبيق.

## نموذج الحماية المقترح (أوفلاين)
استخدم الطبقات التالية معًا:

1. ربط الترخيص بالجهاز
- توليد بصمة جهاز ثابتة نسبيًا machineHash.
- الترخيص يحتوي نفس machineHash.
- الترخيص يعمل فقط على الجهاز المرتبط به.

2. ترخيص موقّع رقميا
- بناء بيانات الترخيص بصيغة JSON.
- توقيع البيانات بالمفتاح الخاص عند البائع فقط.
- التطبيق يتحقق بالمفتاح العام فقط.
- أي تعديل على بيانات الترخيص يفشل التحقق مباشرة.

3. حماية من إرجاع وقت الجهاز
- حفظ آخر وقت تشغيل موثوق.
- عند اكتشاف رجوع غير منطقي في الساعة، يدخل التطبيق وضع حماية.

4. فترة سماح + وضع مقيّد
- قبل الانتهاء: إظهار تنبيهات.
- بعد الانتهاء: فترة سماح اختيارية.
- بعد السماح: وضع قراءة فقط أو تعطيل العمليات الحساسة (البيع/الشراء) مع بقاء عرض البيانات.

## قالب بيانات الترخيص
يمكن استخدام Payload بالشكل التالي:

```json
{
  "licenseId": "LIC-2026-000123",
  "customerId": "CUST-0142",
  "customerName": "Sample Shop",
  "plan": "Annual",
  "issuedAt": "2026-04-11T10:30:00Z",
  "expiresAt": "2027-04-10T23:59:59Z",
  "machineHash": "sha256:...",
  "features": ["sales", "purchases", "reports"],
  "maxTransfersPerYear": 2,
  "appVersionMin": "1.0.0"
}
```

وبجوارها التوقيع:

```json
{
  "payload": { "...": "..." },
  "signature": "base64-signature"
}
```

## مسار التفعيل الكامل بدون إنترنت

### على جهاز العميل
1. تثبيت التطبيق.
2. فتح شاشة التفعيل.
3. التطبيق يعرض Machine Code (مشتق من machineHash).
4. العميل يرسل Machine Code لك.

### عندك أنت
1. فتح أداة توليد التراخيص الداخلية.
2. إدخال بيانات العميل + machineHash + تاريخ الانتهاء + المزايا.
3. توقيع الترخيص بالمفتاح الخاص.
4. إرسال كود/ملف التفعيل للعميل.

### على جهاز العميل مرة أخرى
1. العميل يلصق أو يستورد كود/ملف التفعيل.
2. التطبيق يتحقق من:
- صحة التوقيع.
- تاريخ الصلاحية.
- تطابق machineHash.
3. عند النجاح يتم حفظ حالة التفعيل ويعمل التطبيق بشكل طبيعي.

## سياسة نقل الجهاز
السياسة المقترحة:
- حتى مرتين نقل سنويًا لكل عميل.
- كل عملية نقل تصدر licenseId جديد.
- ترخيص الجهاز القديم يعتبر ملغى حسب السياسة.

إجراءات النقل:
1. العميل يطلب النقل.
2. تجمع Machine Code القديم والجديد.
3. تصدر ترخيصًا جديدًا موقّعًا للجهاز الجديد.
4. تحفظ سجل النقل في ملف المتابعة لديك.

## نقاط الدمج داخل المشروع

1. بوابة ترخيص عند بدء التطبيق
- أضف فحص الترخيص قبل واجهة التطبيق الأساسية.
- نقطة الدخول: lib/main.dart.
- المسار: تهيئة اللغة والخدمات ثم فحص الترخيص ثم التوجيه إلى ActivationPage أو التطبيق.

2. تسجيل خدمة الترخيص
- إضافة LicenseService داخل service locator.
- الملف: lib/services/di/service_locator.dart.

3. الاعتماد على تسجيل الدخول
- اجعل تسجيل الدخول تجربة استخدام فقط.
- قرار السماح الحقيقي يجب أن يعتمد على صلاحية الترخيص، وليس بيانات ثابتة.

4. معرفة مسار البيانات
- قاعدة SQLite تُنشأ داخل Application Support Directory.
- الملف: lib/services/database/app_database.dart.

## التشغيل الفعلي لنظام الحماية (مرة واحدة قبل أول عميل)
1. توليد زوج مفاتيح:

```bash
dart run tool/generate_license_keypair.dart
```

2. احتفظ بقيمة PRIVATE_KEY_BASE64 في مكان آمن جدًا ولا تضعها داخل التطبيق.
3. انسخ قيمة PUBLIC_KEY_BASE64 وضعها بدل القيمة الثابتة داخل:
- lib/features/license/domain/license_service.dart
4. أعد بناء التطبيق بعد تحديث المفتاح العام.

## إصدار كود تفعيل للعميل
بعد ما العميل يرسل لك Machine Hash من شاشة التفعيل، شغّل:

```bash
$env:PRIVATE_KEY_BASE64="ضع-المفتاح-الخاص-هنا"
dart run tool/generate_license.dart <machineHash> <customerName> <licenseId> <expiresAtIsoUtc>
```

مثال:

```bash
$env:PRIVATE_KEY_BASE64="..."
dart run tool/generate_license.dart 9a1f... ShopA LIC-2026-001 2027-04-10T23:59:59Z
```

الناتج يكون Activation Code بصيغة JSON. انسخه للعميل ليضعه في شاشة التفعيل.

## تثبيت وتشغيل العميل لأول مرة (خطوة بخطوة)

1. انسخ مجلد Release إلى جهاز العميل.
2. شغّل ملف التطبيق التنفيذي.
3. إذا ظهرت شاشة التفعيل:
- انسخ Machine Code.
- أرسله لك كالبائع.
4. استقبل كود/ملف التفعيل.
5. ألصق أو استورد التفعيل داخل التطبيق.
6. تأكد أن الحالة تظهر:
- Active.
- تاريخ الانتهاء.
- اسم العميل المرخّص له.
7. نفذ اختبار سريع:
- إضافة منتج واحد.
- إنشاء فاتورة بيع واحدة.
- إغلاق وفتح التطبيق والتأكد من بقاء البيانات.

## دليل التشغيل لك أنت (البائع)

1. قبل التسليم
- بناء نسخة Release.
- التحقق من التحليل والاختبارات.
- تجربة التفعيل على جهاز نظيف.

2. وقت التسليم
- تثبيت التطبيق على جهاز العميل.
- إكمال التفعيل باستخدام Machine Code الحقيقي.
- حفظ لقطة شاشة لحالة الترخيص النشطة.
- تسجيل licenseId وتاريخ الانتهاء في ملف المتابعة.

3. التجديد
- تنبيه العميل قبل الانتهاء بـ 30 يومًا.
- إصدار ترخيص تجديد بتاريخ انتهاء جديد.
- العميل يستورد التجديد أوفلاين.

4. مشاكل الدعم الشائعة
- تلف ملف الترخيص: إعادة إصدار نفس الترخيص.
- تغيير الجهاز: تطبيق سياسة النقل وإصدار ترخيص جديد.
- الاشتباه في التلاعب بالوقت: ضبط وقت الجهاز الصحيح ثم إعادة التحقق.

## إعدادات افتراضية مقترحة
- مدة الترخيص: 12 شهرًا.
- التنبيهات قبل الانتهاء: 30 و15 و7 أيام.
- فترة السماح: 7 أيام.
- بعد السماح: حظر عمليات الكتابة مع السماح بالعرض والتقارير والتصدير.

## ملاحظات عملية مهمة
- لا يوجد نظام أوفلاين غير قابل للكسر بنسبة 100%.
- الهدف هو رفع تكلفة الاختراق وتحسين التحكم التشغيلي.
- لا تضع المفتاح الخاص داخل التطبيق نهائيًا.
- احتفظ بالمفتاح الخاص في بيئة داخلية آمنة لديك فقط.

## قائمة تحقق داخلية لكل عميل
- إنشاء ملف العميل.
- استلام Machine Code.
- إصدار ترخيص موقّع.
- تأكيد نجاح التفعيل على الجهاز المستهدف.
- تسجيل تاريخ الانتهاء وعدد مرات النقل.
- مشاركة قناة الدعم مع العميل.

## المراحل التنفيذية القادمة

1. تنفيذ خدمة Machine Fingerprint.
2. تنفيذ التحقق من التوقيع داخل التطبيق.
3. بناء شاشة التفعيل وبوابة البدء.
4. إزالة الاعتماد على بيانات الدخول التجريبية في تدفق الإنتاج.
5. إضافة صفحة حالة الترخيص ضمن الإعدادات.
6. إضافة قيود الوضع المقيّد على عمليات البيع والشراء الكتابية.
